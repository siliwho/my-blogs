# GCC Extension Keywords & Compiler Builtins
## The Complete Reference for Firmware, Kernel & Systems Engineers

> All keywords prefixed with `__` are GCC/Clang extensions — they are **not** standard C.
> The `__keyword__` form (double underscore on both sides) is the safe spelling: it works
> even when the user has defined a macro named `keyword`. All examples compile with
> `gcc -std=c11` or `clang -std=c11` unless otherwise noted.

---

## 1. `#pragma` — Compiler Directives

`#pragma` is a standardised escape hatch for compiler-specific instructions. Unknown pragmas are ignored (no error), making them safe to use in portable code guarded by `#ifdef`.

### 1.1 `#pragma once`

```c
// Instead of the traditional header guard:
#ifndef MY_HEADER_H
#define MY_HEADER_H
// ...
#endif

// Use:
#pragma once   // supported by GCC, Clang, MSVC, IAR, Keil — effectively universal
```

**How it works:** The compiler tracks the file's inode/device pair, not just its name. Immune to path aliasing issues that can trip up traditional header guards in some build systems.

**Kernel note:** The Linux kernel does NOT use `#pragma once` — it uses traditional guards for strict POSIX portability. Your own code can freely use it.

---

### 1.2 `#pragma GCC optimize`

Override optimisation level per-function or per-file:

```c
// Disable optimisation for a specific function (e.g., a precise timing loop)
#pragma GCC optimize("O0")
void precise_delay_us(uint32_t us) {
    volatile uint32_t count = us * CYCLES_PER_US;
    while (count--);
}
#pragma GCC optimize("O2")   // restore

// Or use __attribute__((optimize)) — cleaner, per-function:
__attribute__((optimize("O0")))
void precise_delay_us(uint32_t us) { ... }
```

---

### 1.3 `#pragma GCC diagnostic`

Suppress specific warnings locally without disabling them globally:

```c
#pragma GCC diagnostic push
#pragma GCC diagnostic ignored "-Wunused-parameter"
#pragma GCC diagnostic ignored "-Wsign-compare"

void legacy_callback(int event, void *data, int unused_param) {
    // old code with intentional unused param
    process(event, data);
}

#pragma GCC diagnostic pop   // restore previous warning state
```

Clang equivalent — same syntax, different warning names:
```c
#pragma clang diagnostic push
#pragma clang diagnostic ignored "-Wdeprecated-declarations"
// ... use deprecated API ...
#pragma clang diagnostic pop
```

---

### 1.4 `#pragma pack`

Control struct padding — see the Structs blog for the full story. Quick reference:

```c
#pragma pack(push, 1)   // save current alignment, set to 1-byte
typedef struct {
    uint8_t  cmd;       // offset 0
    uint16_t length;    // offset 1  (would normally be offset 2)
    uint32_t crc;       // offset 3  (would normally be offset 4)
} __attribute__((packed)) wire_frame_t;   // GCC style — prefer this
#pragma pack(pop)       // restore
```

**Prefer `__attribute__((packed))`** over `#pragma pack` — it is per-type, not file-scope, so it cannot accidentally affect unrelated structs.

---

### 1.5 `#pragma GCC poison`

Permanently ban a symbol — any use after this point is a compile error:

```c
// In a safety-critical firmware codebase:
#pragma GCC poison malloc free realloc calloc
// Any use of malloc anywhere below here → compile error
// Enforces "no dynamic allocation after init" without a linter
```

Extremely useful in safety-critical (MISRA, DO-178C) firmware where dynamic allocation is forbidden.

---

### 1.6 `#pragma message` / `#pragma GCC warning` / `#pragma GCC error`

```c
#pragma message "Building for target: " STRINGIFY(TARGET_MCU)

#if BAUD_RATE > 921600
#pragma GCC warning "Baud rate may be unreliable on this clock configuration"
#endif

#ifndef CRYPTO_KEY_SIZE
#pragma GCC error "CRYPTO_KEY_SIZE must be defined"
#endif
```

---

### 1.7 `_Pragma` — Programmable Pragma (C99)

Allows pragmas to be generated by macros (you cannot put `#pragma` inside a macro expansion):

```c
#define DO_PRAGMA(x)    _Pragma(#x)
#define IGNORE_WARNING(w) \
    DO_PRAGMA(GCC diagnostic push) \
    DO_PRAGMA(GCC diagnostic ignored #w)
#define RESTORE_WARNING() \
    DO_PRAGMA(GCC diagnostic pop)

IGNORE_WARNING(-Wunused-variable)
int legacy_result = old_api();   // no warning
RESTORE_WARNING()
```

## 2. Predefined Identifier Macros

### 2.1 `__func__` (C99 standard)

A predefined identifier (not a macro — a `static const char[]`) containing the **unqualified** function name.

```c
void uart_write(const char *data, size_t len) {
    if (!data) {
        log_error("[%s] null pointer\n", __func__);
        // prints: [uart_write] null pointer
        return;
    }
}
```

**Memory location:** `.rodata` — the string is stored once in the binary.
**Type:** `static const char __func__[]` — implicitly declared in every function body.

---

### 2.2 `__FUNCTION__` (GCC extension)

Identical to `__func__` in GCC/Clang. A GCC-ism predating C99. Prefer `__func__` in new code — it is standard C99.

```c
// These are identical on GCC/Clang:
printf("%s\n", __func__);
printf("%s\n", __FUNCTION__);
```

---

### 2.3 `__PRETTY_FUNCTION__` (GCC extension)

The **decorated** function name — includes return type, class/namespace (in C++), and full parameter types. In plain C, adds the return type.

```c
// In C:
void foo(int x, const char *s) {
    printf("%s\n", __PRETTY_FUNCTION__);
    // output: void foo(int, const char *)
}

// In C++:
namespace hal {
    template<typename T>
    class Driver {
        void read(T *buf, size_t n) {
            printf("%s\n", __PRETTY_FUNCTION__);
            // output: void hal::Driver<T>::read(T*, size_t) [with T = uint8_t]
        }
    };
}
```

**Firmware/kernel logging pattern:**
```c
#define LOG_ERR(fmt, ...) \
    printk(KERN_ERR "[%s:%d] " fmt "\n", __func__, __LINE__, ##__VA_ARGS__)

#define ASSERT(cond) \
    do { \
        if (!(cond)) { \
            panic("ASSERT FAILED: %s — %s:%d in %s\n", \
                  #cond, __FILE__, __LINE__, __PRETTY_FUNCTION__); \
        } \
    } while (0)
```

---

### 2.4 Other Predefined Macros

| Macro              | Type     | Content                                      |
|--------------------|----------|----------------------------------------------|
| `__FILE__`         | Standard | Source file path as a string literal         |
| `__LINE__`         | Standard | Current line number as an integer            |
| `__DATE__`         | Standard | Compilation date `"Mmm dd yyyy"`            |
| `__TIME__`         | Standard | Compilation time `"hh:mm:ss"`               |
| `__STDC_VERSION__` | Standard | `201112L` for C11, `199901L` for C99        |
| `__GNUC__`         | GCC/Clang| GCC major version (Clang emulates this)      |
| `__clang__`        | Clang    | Defined only on Clang                        |
| `__cplusplus`      | C++      | Defined only when compiling as C++           |
| `__COUNTER__`      | GCC/Clang| Unique integer, increments each expansion    |
| `__BASE_FILE__`    | GCC      | Top-level translation unit file              |
| `__INCLUDE_LEVEL__`| GCC      | Current nesting depth of `#include`          |

```c
// __COUNTER__ — generating unique names in macros (e.g., for Linux list_head tricks)
#define CONCAT_(a, b) a##b
#define CONCAT(a, b)  CONCAT_(a, b)
#define UNIQUE_VAR(prefix) CONCAT(prefix, __COUNTER__)

int UNIQUE_VAR(tmp) = 0;   // expands to: int tmp0 = 0;
int UNIQUE_VAR(tmp) = 1;   // expands to: int tmp1 = 1;
```

## 3. `__attribute__` — The Swiss Army Knife

`__attribute__((name))` applies attributes to functions, variables, types, and labels. Multiple attributes can be combined:

```c
__attribute__((noinline, cold, unused))
static void debug_dump(void) { ... }
```

The `__attribute__` and `__attribute__` spellings are identical.

### 3.1 Function Attributes

#### `noreturn`
```c
__attribute__((noreturn))
void panic(const char *msg) {
    puts(msg);
    for (;;);        // or: abort(), exit(), longjmp()
}
// Compiler knows control never returns → no "missing return" warnings
// → better dead-code elimination after calls to panic()
```

#### `noinline` / `always_inline`
```c
__attribute__((noinline))
void trace_point(void) { ... }    // never inlined — stays as a call site for profiling

__attribute__((always_inline))
static inline uint32_t read_reg(volatile uint32_t *r) { return *r; }
// Force inline even at -O0. Error if inlining is impossible.
```

#### `pure` / `const`
```c
// pure: return value depends only on args + global memory. No side effects.
__attribute__((pure))
int strlen_pure(const char *s);    // may read memory, but doesn't write

// const: return value depends ONLY on args. No memory access at all.
__attribute__((const))
int square(int x) { return x * x; }   // compiler may call this once and reuse result
```

#### `cold` / `hot`
```c
__attribute__((cold))   // rarely executed — placed in .text.cold section, de-prioritised
void error_handler(int code) { ... }

__attribute__((hot))    // frequently executed — aggressively optimised, kept in I-cache
void main_loop_iteration(void) { ... }
```

#### `warn_unused_result`
```c
__attribute__((warn_unused_result))
int read_sensor(uint32_t *out);

// Caller gets a warning if they ignore the return value:
read_sensor(&val);   // ✓
read_sensor(NULL);   // ✓ (maybe intentional)
// read_sensor(&val) alone on a line → warning: ignoring return value
```

#### `deprecated`
```c
__attribute__((deprecated("use read_sensor_v2() instead")))
int read_sensor_v1(void);
// Any call to read_sensor_v1() → warning with the message
```

#### `visibility`
```c
// For shared libraries (.so / .dll):
__attribute__((visibility("default")))   // exported — appears in dynamic symbol table
int public_api(void) { ... }

__attribute__((visibility("hidden")))    // not exported — faster, no PLT indirection
static int internal_helper(void) { ... }
```

Build with `-fvisibility=hidden` to hide all symbols by default, then explicitly mark the public API — standard practice for shared libraries.

#### `constructor` / `destructor`
```c
__attribute__((constructor))           // runs before main()
static void module_init(void) {
    register_driver(&my_driver);
}

__attribute__((destructor))            // runs after main() / exit()
static void module_cleanup(void) {
    deregister_driver(&my_driver);
}

// Priority (lower = earlier constructor, later destructor):
__attribute__((constructor(101)))      // runs very early
static void platform_init(void) { ... }

__attribute__((constructor(200)))      // runs after platform_init
static void subsystem_init(void) { ... }
```

**Kernel equivalent:** `module_init()` / `module_exit()` macros use a similar mechanism with linker sections.

#### `format`
```c
// Tell the compiler this function uses printf-style format strings:
__attribute__((format(printf, 1, 2)))
void log_printf(const char *fmt, ...);
// Now the compiler type-checks format string vs arguments:
log_printf("%d %s", 42, "hello");   // ✓
log_printf("%d", "wrong type");      // warning: format mismatch
```

#### `nonnull`
```c
__attribute__((nonnull(1, 3)))       // args 1 and 3 must not be NULL
int process(void *input, int flags, void *output);
// Passing NULL for those args → warning at call site
```

#### `sentinel`
```c
__attribute__((sentinel))
void exec_args(const char *prog, ...);
// Last argument in the variadic list must be NULL (sentinel):
exec_args("ls", "-la", NULL);    // ✓
exec_args("ls", "-la");          // warning: missing NULL sentinel
```

#### `weak` / `alias`
```c
// Weak: overrideable default implementation
__attribute__((weak))
void SysTick_Handler(void) { }    // default no-op; user can override

// Alias: another name for the same function
void uart0_handler(void);
__attribute__((alias("uart0_handler")))
void USART0_IRQHandler(void);     // both names refer to the same code
```

#### `interrupt` / `signal` (embedded)
```c
// Cortex-M ISRs are normal C functions — no special attribute needed
// AVR ISRs:
#include <avr/interrupt.h>
ISR(TIMER0_OVF_vect) { ... }

// x86 interrupt gate (used in OS kernels):
__attribute__((interrupt))
void keyboard_isr(struct interrupt_frame *frame) { ... }
```

### 3.2 Variable / Object Attributes

#### `aligned`
```c
// Force alignment to N bytes (N must be power of 2):
uint8_t dma_buf[1024] __attribute__((aligned(32)));   // 32-byte aligned for cache lines

// Useful for SIMD (SSE requires 16-byte, AVX requires 32-byte):
float vectors[256] __attribute__((aligned(32)));

// Structs:
struct per_cpu_data {
    int value;
    char _pad[60];
} __attribute__((aligned(64)));   // one struct per cache line — prevents false sharing

// C11 portable equivalent:
_Alignas(64) struct per_cpu_data cpu_data[MAX_CPUS];
```

#### `packed`
```c
typedef struct {
    uint8_t  cmd;
    uint32_t addr;
    uint16_t len;
} __attribute__((packed)) spi_cmd_t;
// sizeof = 7, not 8 — no padding inserted
```

#### `section`
```c
// Place in named linker section:
__attribute__((section(".isr_vector")))
const uint32_t vector_table[] = { ... };

__attribute__((section(".ccmram")))   // STM32 Core-Coupled Memory — fast, no DMA
uint8_t fast_buf[256];

__attribute__((section(".noinit")))   // survives warm reset — not zeroed by startup
uint32_t reset_reason;

// Linux kernel equivalent macros:
// __init → .init.text (freed after boot)
// __initdata → .init.data
// __exit → .exit.text
```

#### `used` / `unused`
```c
__attribute__((used))
static const char build_date[] = __DATE__;   // not referenced in code, but kept in binary

__attribute__((unused))
static void debug_helper(void) { ... }      // suppress "defined but not used" warning
```

#### `cleanup`
```c
// Run a function when the variable goes out of scope — RAII in C:
void close_fd(int *fd) {
    if (*fd >= 0) close(*fd);
}

void process_file(const char *path) {
    int fd __attribute__((cleanup(close_fd))) = open(path, O_RDONLY);
    if (fd < 0) return;   // fd automatically closed on every return path
    // ... use fd ...
}   // fd closed here automatically
```

The Linux kernel uses `__cleanup()` (a wrapper around this) extensively since kernel 6.x for lock auto-release.

#### `transparent_union`
```c
// Used in glibc to make a union accept multiple pointer types:
typedef union {
    int     *__ip;
    long    *__lp;
    void    *__vp;
} __attribute__((transparent_union)) param_t;

void syscall_wrapper(param_t p);
// Can be called with int*, long*, or void* — no cast needed
```

---

### 3.3 Type Attributes

```c
// May be applied to struct/union/enum:
typedef struct {
    int x, y;
} __attribute__((packed, aligned(4))) point_t;

// Deprecated type:
typedef unsigned long __attribute__((deprecated)) ulong_compat;
```

## 4. `__alignof` / `__alignof__` / `_Alignof`

Returns the **alignment requirement** of a type or expression in bytes.

```c
__alignof__(char)      == 1
__alignof__(short)     == 2
__alignof__(int)       == 4
__alignof__(double)    == 8
__alignof__(void *)    == 8   // on 64-bit

// Also works on expressions:
int x;
__alignof__(x)         == 4

// C11 standard spelling:
_Alignof(double)       == 8
```

**Practical uses:**
```c
// Verify DMA buffer alignment at compile time:
static_assert(__alignof__(dma_buf) >= 32, "DMA buffer must be 32-byte aligned");

// Generic allocator that respects type alignment:
#define ALLOC_ALIGNED(type, count) \
    aligned_alloc(__alignof__(type), sizeof(type) * (count))

// Check if a pointer is aligned before a SIMD operation:
static inline bool is_aligned(const void *p, size_t align) {
    return ((uintptr_t)p & (align - 1)) == 0;
}
if (!is_aligned(buf, __alignof__(uint64_t))) {
    // use scalar fallback
}
```

## 5. `__asm__` / `__asm` — Inline Assembly

`__asm__` and `__asm` are synonyms for `asm`. The double-underscore form works even in strict C modes (`-std=c11`) where the plain `asm` keyword is not available.

### 5.1 Basic Syntax

```c
__asm__ [volatile] ( "instructions"
    : output operands
    : input operands
    : clobbers
);
```

### 5.2 Simple Inline Assembly

```c
// NOP — do nothing for one cycle (timing, pipeline flush)
__asm__ __volatile__("nop");

// Memory barrier — prevent compiler and CPU reordering:
__asm__ __volatile__("" ::: "memory");   // compiler barrier only
__asm__ __volatile__("mfence" ::: "memory");   // x86 full memory fence

// ARM data memory barrier:
__asm__ __volatile__("dmb sy" ::: "memory");

// ARM instruction synchronisation barrier (required after modifying code):
__asm__ __volatile__("isb" ::: "memory");
```

### 5.3 Operands and Constraints

```c
// Read the x86 timestamp counter (RDTSC):
uint64_t rdtsc(void) {
    uint32_t lo, hi;
    __asm__ __volatile__(
        "rdtsc"
        : "=a"(lo), "=d"(hi)   // output: lo → EAX, hi → EDX
        :                        // no inputs
        :                        // no clobbers (EAX, EDX captured as outputs)
    );
    return ((uint64_t)hi << 32) | lo;
}

// ARM: read a system register (CPUID):
uint32_t read_midr(void) {
    uint32_t val;
    __asm__ __volatile__(
        "mrc p15, 0, %0, c0, c0, 0"   // read MIDR into %0
        : "=r"(val)
    );
    return val;
}

// x86: atomically exchange a value:
uint32_t atomic_xchg(volatile uint32_t *ptr, uint32_t val) {
    __asm__ __volatile__(
        "xchgl %0, %1"
        : "+r"(val), "+m"(*ptr)   // +: read-write
        :
        : "memory"
    );
    return val;
}
```

### 5.4 Common Constraints

| Constraint | Meaning |
|-----------|---------|
| `"r"`     | Any general-purpose register |
| `"m"`     | Memory operand |
| `"i"`     | Immediate integer constant |
| `"n"`     | Immediate known at compile time |
| `"a"`     | EAX / RAX |
| `"b"`     | EBX / RBX |
| `"c"`     | ECX / RCX |
| `"d"`     | EDX / RDX |
| `"S"`     | ESI / RSI |
| `"D"`     | EDI / RDI |
| `"="`     | Output (write only) |
| `"+"`     | Read-write operand |
| `"&"`     | Early clobber (written before inputs are used) |

### 5.5 `__volatile__` on `asm`

`__asm__ __volatile__` prevents the compiler from:
- Removing the asm if its outputs are unused
- Moving the asm across other operations
- Hoisting it out of loops

**Always use `__volatile__` for:**
- I/O operations (`in`/`out` on x86)
- Barrier instructions (`mfence`, `dmb`, `dsb`, `isb`)
- Instructions with side effects (`wfi`, `wfe`, `sev`, `rdtsc`)
- Any asm that reads/writes hardware state

```c
// WFI (Wait For Interrupt) — ARM power saving
__attribute__((always_inline))
static inline void cpu_wfi(void) {
    __asm__ __volatile__("wfi" ::: "memory");
}
```

### 5.6 Naked Functions

```c
// No prologue/epilogue generated — you write the entire function in asm:
__attribute__((naked))
void Reset_Handler(void) {
    __asm__ __volatile__(
        "ldr  sp, =_estack      \n"
        "bl   SystemInit        \n"
        "bl   __libc_init_array \n"
        "bl   main              \n"
        "b    .                 \n"   // infinite loop if main returns
    );
}
```

Used for reset handlers, context switchers, and syscall stubs in kernels.

## 6. `__typeof__` / `__typeof`

Yields the type of an expression — the GCC precursor to C++11 `decltype`. Used extensively in the Linux kernel.

```c
int x = 5;
__typeof__(x) y = 10;          // y is int

double d = 3.14;
__typeof__(d) *ptr = &d;       // ptr is double *

// Key use: type-safe macros that avoid double-evaluation:
#define MAX(a, b) ({                    \
    __typeof__(a) _a = (a);             \
    __typeof__(b) _b = (b);             \
    _a > _b ? _a : _b;                  \
})

// Compare with the broken macro version:
#define BAD_MAX(a,b)  ((a) > (b) ? (a) : (b))
// BAD_MAX(i++, j++) → i or j incremented TWICE

int result = MAX(foo(), bar());   // foo() and bar() called exactly once
```

**Linux kernel uses `__typeof__` everywhere:**
```c
// From include/linux/kernel.h:
#define swap(a, b) \
    do { __typeof__(a) __tmp = (a); (a) = (b); (b) = __tmp; } while (0)

// container_of with type checking:
#define container_of(ptr, type, member) ({              \
    void *__mptr = (void *)(ptr);                       \
    static_assert(__same_type(*(ptr), ((type *)0)->member) || \
                  __same_type(*(ptr), void),            \
                  "pointer type mismatch in container_of()"); \
    ((type *)(__mptr - offsetof(type, member))); })
```

## 7. `__builtin_offsetof`

Computes the byte offset of a member within a struct — identical to `offsetof()` from `<stddef.h>`, but this is the compiler builtin that `offsetof` typically expands to.

```c
#include <stddef.h>

struct device {
    int      id;       // offset 0
    char     name[32]; // offset 4
    uint32_t flags;    // offset 36
};

size_t off = __builtin_offsetof(struct device, flags);  // == 36
// Equivalent to:
size_t off = offsetof(struct device, flags);

// Use case: container_of pointer arithmetic
struct device *dev_from_flags(uint32_t *flags_ptr) {
    return (struct device *)((char *)flags_ptr
           - __builtin_offsetof(struct device, flags));
}

// Compile-time layout verification:
_Static_assert(__builtin_offsetof(struct device, flags) == 36,
               "flags field moved — check serialisation code");
```

## 8. `__builtin_va_arg` and Variadic Builtins

These are the low-level primitives behind `<stdarg.h>`. Normally you use the standard macros, but understanding the builtins matters for implementing custom printf-style functions.

```c
#include <stdarg.h>

// Standard usage (backed by GCC builtins):
void my_log(const char *fmt, ...) {
    va_list args;
    va_start(args, fmt);                    // __builtin_va_start
    vprintf(fmt, args);                     // passes va_list to vprintf
    va_end(args);                           // __builtin_va_end
}

// Forwarding va_list — MUST copy if passing to multiple functions:
void my_log2(const char *fmt, va_list args) {
    va_list args_copy;
    va_copy(args_copy, args);               // __builtin_va_copy
    int needed = vsnprintf(NULL, 0, fmt, args_copy);   // measure
    va_end(args_copy);
    char *buf = malloc(needed + 1);
    vsnprintf(buf, needed + 1, fmt, args);             // write
    send_log(buf);
    free(buf);
}
```

**Direct builtin — rarely needed but good to know:**
```c
void dump_ints(int count, ...) {
    __builtin_va_list ap;
    __builtin_va_start(ap, count);
    for (int i = 0; i < count; i++) {
        int val = __builtin_va_arg(ap, int);
        printf("%d ", val);
    }
    __builtin_va_end(ap);
}
```

## 9. `__extension__`

Suppresses the `-pedantic` warning that GCC/Clang emit when a GCC extension is used in strict standard mode:

```c
// Compiling with -std=c99 -pedantic, using a GCC extension:
__extension__ typedef long long ll_t;   // no pedantic warning
__extension__ int x = 0b10110101;       // binary literals (GCC extension) — no warning
```

This is primarily used in system headers (`<stdint.h>`, `<sys/types.h>`) to prevent noise when included with `-pedantic`. In your own code, just compile without `-pedantic` or accept the warnings.

## 10. `__restrict` / `__restrict__`

The GCC spellings of the C99 `restrict` keyword. Tells the compiler that a pointer is the **only** pointer that accesses that memory region — enabling alias analysis and vectorisation.

```c
// Standard C99:
void add_arrays(int * restrict dst,
                const int * restrict src1,
                const int * restrict src2,
                size_t n);

// GCC extension spellings (work in C89 mode too):
void add_arrays(int *__restrict__ dst,
                const int *__restrict__ src1,
                const int *__restrict__ src2,
                size_t n);
```

**What the compiler does with `restrict`:**
- Assumes `dst`, `src1`, `src2` do not alias each other
- Can auto-vectorise the loop (generate SIMD instructions)
- Can reorder loads and stores freely within the function
- Violating `restrict` (actually aliasing) is **undefined behaviour**

**Real performance impact:**
```c
// Without restrict — compiler must assume dst might alias src:
void copy(float *dst, const float *src, size_t n) {
    for (size_t i = 0; i < n; i++) dst[i] = src[i];
    // generates scalar loop — cannot vectorise safely
}

// With restrict — compiler generates AVX/NEON vectorised loop:
void copy(float *__restrict__ dst, const float *__restrict__ src, size_t n) {
    for (size_t i = 0; i < n; i++) dst[i] = src[i];
    // 8× or 16× speedup with AVX2 / NEON
}
```

**`memcpy` is declared with restrict in glibc:**
```c
void *memcpy(void *__restrict__ dest, const void *__restrict__ src, size_t n);
// This is why memcpy is undefined for overlapping regions — use memmove instead.
```

## 11. `__signed__` / `__signed`

The GCC spelling of the `signed` keyword — used in system headers to avoid conflicts with user-defined macros named `signed`:

```c
// In <sys/types.h> and kernel headers:
typedef __signed__ int  __s32;
typedef __signed__ char __s8;

// In user code, just use:
signed int x;
int8_t y;   // prefer <stdint.h>
```

You will rarely write this yourself, but you will see it in glibc and kernel headers. It means exactly `signed`.

---

## 12. `__const`

The GCC spelling of `const` — same purpose as `__signed__`: prevents conflicts with user macros that redefine `const`.

```c
// In system headers:
extern __const char *__const sys_errlist[];

// Equivalent to:
extern const char *const sys_errlist[];
```

Again, you will see this in glibc headers. In your own code, use `const`.

---

## 13. `__volatile__` / `__volatile`

The GCC spelling of `volatile`. Identical behaviour — exists to survive `-pedantic` mode and macro namespace pollution.

```c
// These are all identical:
volatile uint32_t *reg = (volatile uint32_t *)0x40000000;
__volatile__ uint32_t *reg = (__volatile__ uint32_t *)0x40000000;

// In inline assembly — both forms are identical:
__asm__ __volatile__("nop");
asm volatile("nop");
```

In practice, always write `volatile` in application code. Use `__volatile__` only inside macros that may be included in strict-mode builds.

## 14. `__inline__` / `__inline`

The GCC spelling of `inline`. Works in C89 mode where `inline` is not a keyword.

```c
// Standard C99/C11:
static inline uint32_t bswap32(uint32_t x) {
    return __builtin_bswap32(x);
}

// GCC extension spelling (C89 compatible):
static __inline__ uint32_t bswap32(uint32_t x) {
    return __builtin_bswap32(x);
}
```

For cross-compiler portability macros:
```c
#if defined(__GNUC__) || defined(__clang__)
#  define FORCE_INLINE  __attribute__((always_inline)) static __inline__
#  define NO_INLINE     __attribute__((noinline))
#elif defined(_MSC_VER)
#  define FORCE_INLINE  __forceinline
#  define NO_INLINE     __declspec(noinline)
#else
#  define FORCE_INLINE  static inline
#  define NO_INLINE
#endif

FORCE_INLINE uint32_t read32(const void *p) {
    uint32_t v; memcpy(&v, p, 4); return v;
}
```

## 15. `__label__` — Local Labels

Allows declaring labels as local to a block — prevents `goto` from jumping into the block from outside. Also enables label-valued expressions when combined with `&&label`.

```c
// Local label declaration:
void state_machine(void) {
    __label__ retry, done;

    retry:
        if (!try_acquire()) goto retry;
        // ... do work ...
    done:
        release();
}

// Label as a value — computed goto (GCC extension):
void *dispatch_table[] = { &&handle_a, &&handle_b, &&handle_c };

int dispatch(int cmd) {
    goto *dispatch_table[cmd];   // jump to label address

    handle_a: return process_a();
    handle_b: return process_b();
    handle_c: return process_c();
}
```

**Computed goto is used in:**
- CPython's bytecode interpreter (huge performance win over switch)
- Lua's VM
- Some kernel fast-path dispatch tables

## 16. `__thread` / `_Thread_local`

Declares **thread-local storage (TLS)** — each thread gets its own copy of the variable.

```c
// GCC extension spelling:
__thread int errno_copy;
__thread struct per_thread_state state;

// C11 standard spelling (prefer this):
_Thread_local int errno_val;

// POSIX macro (works everywhere):
#include <threads.h>
thread_local int counter;    // C11 <threads.h>
```

**How TLS works:**
- Static TLS: compiler allocates fixed offsets in the TLS block; very fast (single register-relative load on x86: `mov %fs:offset, %eax`)
- Dynamic TLS: `__thread` in shared libraries loaded with `dlopen` — uses `pthread_getspecific` internally, slower

**errno is implemented as TLS:**
```c
// In <errno.h>:
extern __thread int errno;    // or _Thread_local — each thread has its own errno
// This is why errno is thread-safe without explicit locking.
```

**Kernel TLS equivalent:**
The Linux kernel uses `DEFINE_PER_CPU(type, name)` for per-CPU variables, which maps to a segment register trick on x86:
```c
DEFINE_PER_CPU(unsigned long, cpu_current_top_of_stack);
per_cpu(cpu_current_top_of_stack, smp_processor_id()) = ...;
```

**Restrictions on `__thread` variables:**
- Can only be zero-initialised or initialised with a constant expression
- Cannot have a non-trivial constructor/destructor (C++ restriction)
- Taking the address gives the current thread's copy

## 17. `__complex__` / `__complex` / `__real__` / `__imag__`

GCC complex number extensions — predating C99's `_Complex`.

```c
// C99 standard:
#include <complex.h>
double complex z = 3.0 + 4.0 * I;

// GCC extension spellings:
__complex__ double z = 3.0 + 4.0i;   // 4.0i is a GCC imaginary literal
double __real__ z_re = __real__ z;    // extract real part
double __imag__ z_im = __imag__ z;   // extract imaginary part
```

In systems code, complex numbers appear in DSP firmware (FFT, filters, IQ baseband processing):
```c
// DSP: multiply two IQ samples
__complex__ float iq_multiply(__complex__ float a, __complex__ float b) {
    return a * b;   // compiler generates FMA or VCMUL instruction
}

// Extract I and Q channels:
float i_sample = __real__ iq_sample;
float q_sample = __imag__ iq_sample;
```

Prefer `_Complex` and `<complex.h>` in new code — it is C99 standard.

## 18. `__null`

The GCC internal representation of `NULL`. In C++ mode, `__null` is a zero-valued pointer constant of the correct pointer type (unlike `0` or `(void*)0` which require casts). In C mode it is identical to `(void*)0`.

```c
// You will see this in system headers — never write it in application code.
// Just use NULL from <stddef.h>:
#define NULL  __null      // in GCC's C++ headers
```

In C11, use `nullptr` in C++ and `NULL` in C. In C23, `nullptr` is available in C too.

## 19. Compiler Builtins Reference

GCC and Clang provide hundreds of `__builtin_*` functions that map to efficient instructions or provide information to the optimiser.

### 19.1 Branch Prediction

```c
// Tell the compiler which branch is expected:
if (__builtin_expect(error_code != 0, 0)) {
    // unlikely path — 0 means "expect condition to be false"
    handle_error();
}

if (__builtin_expect(data_ready, 1)) {
    // likely path — 1 means "expect condition to be true"
    process();
}

// Cleaner macros (used throughout Linux kernel):
#define likely(x)   __builtin_expect(!!(x), 1)
#define unlikely(x) __builtin_expect(!!(x), 0)

// Usage:
if (unlikely(ptr == NULL)) return -EINVAL;
if (likely(cache_hit))     return cache_value;
```

**Real effect:** The compiler lays out the likely path as fall-through (no branch), the unlikely path as taken-branch — better instruction cache behaviour.

### 19.2 Bit Operations

```c
// Count Leading Zeros (CLZ):
__builtin_clz(0x00000001)    == 31   // on 32-bit int
__builtin_clzl(x)                    // long version
__builtin_clzll(x)                   // long long version

// Count Trailing Zeros (CTZ):
__builtin_ctz(0x00000100)    == 8

// Population Count (POPCOUNT) — count set bits:
__builtin_popcount(0xFF)     == 8
__builtin_popcountll(x)              // 64-bit version

// Find First Set bit (1-indexed, 0 if input is 0):
__builtin_ffs(0x00000100)    == 9    // bit 8, 1-indexed = 9

// Parity (XOR of all bits):
__builtin_parity(0xFF)       == 0    // even parity
__builtin_parity(0x0F)       == 0
__builtin_parity(0x01)       == 1    // odd parity
```

**Practical uses:**
```c
// Fast log2 for power-of-2 sizes:
uint32_t log2_floor(uint32_t x) {
    return 31 - __builtin_clz(x);   // maps to single BSR instruction on x86
}

// Allocator: find free slot in a bitmap:
int find_free_slot(uint32_t bitmap) {
    return __builtin_ctz(~bitmap);   // invert, then find first 0-bit
}

// Check if power of two:
bool is_pow2(uint32_t x) {
    return x && (__builtin_popcount(x) == 1);
}
```

### 19.3 Byte Swap

```c
__builtin_bswap16(0x1234)        == 0x3412
__builtin_bswap32(0x12345678)    == 0x78563412
__builtin_bswap64(x)

// Endianness conversion:
uint32_t host_to_be32(uint32_t x) {
#if __BYTE_ORDER__ == __ORDER_LITTLE_ENDIAN__
    return __builtin_bswap32(x);
#else
    return x;
#endif
}
```

### 19.4 Memory Builtins

```c
// Constant-folding versions — compiler may replace with immediate if size is known:
__builtin_memcpy(dst, src, n);
__builtin_memset(dst, val, n);
__builtin_memcmp(a, b, n);
__builtin_strlen(s);

// These are equivalent to the standard library versions but may be inlined:
// e.g., __builtin_memset(buf, 0, 4) → a single MOV immediate instruction
```

### 19.5 Type Information

```c
// Check if two types are the same (returns 1 or 0):
__builtin_types_compatible_p(int, unsigned int)   // 0 — different
__builtin_types_compatible_p(int, int)            // 1 — same

// Linux kernel uses this for type-safe container_of:
#define __same_type(a, b) __builtin_types_compatible_p(__typeof__(a), __typeof__(b))

// Choose expression based on constant condition (evaluated at compile time):
int x = __builtin_choose_expr(sizeof(int) == 4, fast_path(), slow_path());
// Only the chosen expression is compiled — the other is discarded
```

### 19.6 Arithmetic with Overflow Detection

```c
// Returns 1 if overflow, 0 if no overflow. Result in *res.
int overflow;
unsigned int result;
overflow = __builtin_uadd_overflow(a, b, &result);   // unsigned add
overflow = __builtin_usub_overflow(a, b, &result);   // unsigned sub
overflow = __builtin_umul_overflow(a, b, &result);   // unsigned mul
overflow = __builtin_sadd_overflow(a, b, &result);   // signed add
overflow = __builtin_smul_overflow(a, b, &result);   // signed mul

// Kernel pattern — safe size calculation:
size_t total;
if (__builtin_mul_overflow(count, item_size, &total)) {
    return -EOVERFLOW;
}
void *buf = kmalloc(total, GFP_KERNEL);
```

### 19.7 Miscellaneous Builtins

```c
// Compiler barrier — prevent reordering across this point:
__builtin_ia32_sfence();    // x86 store fence
__builtin_ia32_lfence();    // x86 load fence
__builtin_ia32_mfence();    // x86 full fence

// Prefetch — hint the CPU to load cache line:
__builtin_prefetch(ptr, 0, 3);   // (addr, rw=read, locality=high)
__builtin_prefetch(ptr, 1, 1);   // (addr, rw=write, locality=low)

// Unreachable — tell the compiler this point is never reached:
__builtin_unreachable();
// Equivalent to C23's unreachable() macro.
// Use after switch statements with exhaustive enums:
switch (state) {
    case STATE_A: return a();
    case STATE_B: return b();
    // No default needed — but add:
    default: __builtin_unreachable();
}

// Get return address of current or parent frame:
void *ret_addr = __builtin_return_address(0);   // caller of this function
void *ret_addr2 = __builtin_return_address(1);  // caller's caller

// Stack address:
void *frame = __builtin_frame_address(0);       // current frame pointer
```

## 20. `_Static_assert` / `static_assert` — Compile-Time Checks

Not a GCC extension — C11 standard. But critical for systems code and always used alongside the GCC features above.

```c
// C11:
_Static_assert(sizeof(uint32_t) == 4, "uint32_t must be 4 bytes");
_Static_assert(sizeof(wire_frame_t) == 7, "Wire frame layout changed — update protocol doc");
_Static_assert(offsetof(struct device, irq) == 16, "IRQ field moved — check DTS binding");
_Static_assert(__alignof__(dma_descriptor_t) >= 16, "DMA descriptor must be 16-byte aligned");

// C11 also allows the message to be omitted:
_Static_assert(sizeof(long) == 8);

// <assert.h> provides static_assert as a macro (C11):
#include <assert.h>
static_assert(sizeof(int) == 4, "int must be 32-bit");
```

## 21. Statement Expressions `({ ... })`

A GCC extension allowing a block of statements to be used as an expression. The value of the expression is the last statement in the block.

```c
int result = ({
    int tmp = foo();
    tmp = tmp * 2 + bar();
    tmp;   // ← this is the value of the expression
});

// The canonical type-safe MAX macro:
#define MAX(a, b) ({                \
    __typeof__(a) _a = (a);         \
    __typeof__(b) _b = (b);         \
    _a > _b ? _a : _b;              \
})

// Linux kernel container_of uses this:
#define container_of(ptr, type, member) ({          \
    void *__mptr = (void *)(ptr);                   \
    ((type *)(__mptr - offsetof(type, member)));    \
})
```

Statement expressions do not work in C++ `constexpr` or in constant expression contexts. Use with care in shared C/C++ headers.
